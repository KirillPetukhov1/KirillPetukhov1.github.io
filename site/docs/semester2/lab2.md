# Отчет по лабораторной работе "Численные вычисления и анализ данных с использованием NumPy"

## 1. Цель работы
Целью лабораторной работы является изучение основ работы с библиотекой NumPy для численных вычислений, а также применение библиотек Matplotlib и Seaborn для визуализации данных. В ходе работы необходимо реализовать функции для создания массивов, выполнения векторных и матричных операций, статистического анализа данных и построения графиков, после чего проверить их корректность с помощью тестов.

## 2. Описание задачи
Исходное задание (файл `LR-2-main.py`) содержит шаблоны функций с подробными комментариями и ссылками на документацию. Требуется написать реализацию каждой функции, чтобы она проходила все тесты из файла `test.py`. Тесты покрывают как стандартные случаи, так и граничные ситуации (несовместимые размеры массивов, вырожденные матрицы, отсутствие файла и т.д.). Визуализации должны сохраняться в папку `plots`, данные для загрузки берутся из `data/students_scores.csv`.

## 3. Решение

### 3.1. Создание и обработка массивов
- **`create_vector`** - возвращает одномерный массив от 0 до 9 с помощью `np.arange(10)`.
- **`create_matrix`** - генерирует матрицу 5×5 со случайными числами в интервале [0, 1] через `np.random.rand(5,5)`.
- **`reshape_vector`** - изменяет форму вектора (10,) на матрицу (2,5) используя метод `.reshape(2,5)`.
- **`transpose_matrix`** - возвращает транспонированную матрицу (`.T` или `np.transpose`).

### 3.2. Векторные операции
Все операции реализованы с использованием векторизации NumPy, что позволяет избежать явных циклов и повышает производительность.
- **`vector_add`** - поэлементное сложение двух векторов оператором `+`.
- **`scalar_multiply`** - умножение вектора на число оператором `*`.
- **`elementwise_multiply`** - поэлементное произведение двух массивов оператором `*`.
- **`dot_product`** - скалярное произведение через `np.dot(a, b)`. При несовпадении длин NumPy выбрасывает `ValueError`.

### 3.3. Матричные операции
- **`matrix_multiply`** - матричное умножение через оператор `@` (или `np.matmul`).
- **`matrix_determinant`** - вычисление определителя квадратной матрицы (`np.linalg.det`).
- **`matrix_inverse`** - нахождение обратной матрицы (`np.linalg.inv`). Для вырожденной матрицы генерируется исключение `LinAlgError`.
- **`solve_linear_system`** - решение системы линейных уравнений Ax = b (`np.linalg.solve`). Требует, чтобы матрица A была квадратной и невырожденной.

### 3.4. Статистический анализ
- **`load_dataset`** - загружает CSV-файл с помощью `pandas.read_csv` и преобразует в NumPy-массив.
- **`statistical_analysis`** - рассчитывает основные статистики одномерного массива: среднее, медиану, стандартное отклонение, минимум, максимум, 25-й и 75-й перцентили.
- **`normalize_data`** - выполняет Min-Max нормализацию по формуле `(x - min) / (max - min)`. При одинаковых значениях (max = min) возникает деление на ноль, что даёт `inf` или `nan`.

### 3.5. Визуализация
- **`plot_histogram`** - строит гистограмму распределения оценок по математике (10 бинов от 0 до 100) с помощью `plt.hist`.
![Изображение](lab2_res\histogram.png)
- **`plot_heatmap`** - создаёт тепловую карту корреляции между предметами. Сначала из входной матрицы (строки - ученики, столбцы – предметы) формируется DataFrame с названиями столбцов, затем вычисляется корреляционная матрица `df.corr()`, после чего рисуется `sns.heatmap` с аннотациями.
![Изображение](lab2_res\heatmap.png)
- **`plot_line`** – отображает зависимость оценки от номера студента в виде линейного графика с маркерами.
![Изображение](lab2_res\line.png)

Все функции визуализации принимают параметр `save_image` (по умолчанию `True`) для сохранения изображения в папку `plots`. Используется `plt.savefig` с указанием пути.

## 4. Тестирование

Для проверки корректности реализованных функций используется файл `test.py`, содержащий:
- Базовые тесты, проверяющие возвращаемые значения и типы.
- Расширенные тесты с использованием `pytest.raises` для проверки исключений:
- Несовместимые формы при сложении, умножении, скалярном произведении.
- Неквадратные матрицы для определителя, обратной матрицы и решения СЛАУ.
- Вырожденные матрицы (`LinAlgError`).
- Отсутствие файла (`FileNotFoundError`).
- Граничный случай нормализации (константные данные).
- Тесты визуализаций, которые просто проверяют, что функции не падают.

Все тесты успешно пройдены после реализации.

## 5. Нюансы реализации и особенности

1. **Векторизация вместо циклов** - во всех операциях использованы встроенные функции NumPy, что обеспечивает высокую скорость и читаемость кода.
2. **Обработка исключений** - функции линейной алгебры выбрасывают исключения при некорректных входных данных, что проверяется тестами.
3. **Min-Max нормализация** - при `max = min` формула приводит к делению на ноль. В тесте проверяется, что результат содержит `inf` или `nan`. Это поведение соответствует математическому смыслу (нормализация невозможна).
4. **Пути к файлам** - в коде визуализации используются относительные пути `'code/s2_l2/plots/...'`. При запуске из корня проекта важно, чтобы структура папок совпадала. В основной функции `if __name__ == "__main__"` путь к данным задаётся через `Path('code/s2_l2/data/students_scores.csv')`.
5. **Тепловая карта корреляции** - важно правильно преобразовать входную матрицу (ученики X предметы) в DataFrame, чтобы `df.corr()` вычислил корреляции между предметами (столбцами). Если передать матрицу корреляции напрямую, то `df.corr()` повторно вычислит корреляцию, что приведёт к некорректному результату. Поэтому функция `plot_heatmap` ожидает на вход **матрицу данных**, а не уже готовую корреляционную матрицу. Это соответствует заданию.
6. **Сохранение изображений** - графики сохраняются до вызова `plt.show()`, чтобы избежать пустого файла. Параметр `bbox_inches='tight'` (использован только в одном месте, но можно добавить везде) предотвращает обрезание подписей.
7. **Тестирование визуализаций** - в тестах визуализации вызываются с `save_image=False`, чтобы не создавать файлы при каждом запуске тестов. При необходимости можно добавить проверку наличия файлов.
