# Лабораторная работа №1. Создание и развертывание статического сайта на MkDocs

## Цель
Освоить процесс создания статического сайта с использованием генератора документации MkDocs, организовать структуру портфолио лабораторных работ, изучить базовые принципы работы с Git и GitHub, развернуть сайт на GitHub Pages, а также настроить тему оформления и конфигурационный файл mkdocs.yml.

## Задание
1. Создать публичный репозиторий на GitHub.
2. Настроить GitHub Pages для публикации из каталога `/docs` ветки `main`.
3. Клонировать репозиторий, создать виртуальное окружение, установить MkDocs.
4. Настроить `.gitignore` (исключить виртуальное окружение и служебные файлы).
5. Создать новый сайт командой `mkdocs new source`.
6. Запустить локальный сервер для предпросмотра: `mkdocs serve`.
7. Выполнить сборку сайта в каталог `/docs` корня репозитория: `mkdocs build -d ../docs`.
8. Выбрать и подключить тему оформления (Material for MkDocs).
9. Настроить файл `mkdocs.yml`: название сайта, тема, структура меню (`nav`).
10. Создать структуру страниц:
    - Главная страница (`index.md`)
    - Страница «Об авторе» (`about.md`)
    - Раздел «Лабораторные работы» с подразделами «1 семестр» и «2 семестр»:
        - 1 семестр: лабораторные работы №10 и №11 (отчёты `lab10.md`, `lab11.md`)
        - 2 семестр: лабораторная работа №1 (данный отчёт, `lab1.md`)
11. Обеспечить корректную навигацию по сайту.
12. Выполнить повторную сборку и публикацию на GitHub Pages.

## Код и конфигурация

### Конфигурационный файл `mkdocs.yml`
```yaml
site_name: Портфолио студента ИТМО
site_url: https://username.github.io/portfolio/
theme:
  name: material
  palette:
    primary: teal
    accent: deep purple
  font:
    text: Roboto
    code: Fira Code
  features:
    - navigation.tabs
    - navigation.sections
    - toc.integrate
    - search.suggest
    - search.highlight

nav:
  - Главная: index.md
  - Об авторе: about.md
...
```

### Пример команды для создания структуры
#### Создание нового проекта MkDocs
```bash
mkdocs new site
cd site
```

#### Запуск локального сервера
```bash
mkdocs serve
```

#### Сборка сайта в папку ../docs
```bash
mkdocs build -d ../docs
```

## Выводы
В результате выполнения лабораторной работы был создан статический сайт-портфолио с использованием генератора MkDocs. В первом семестре представлены отчёты по лабораторным работам №10 и №11, во втором – отчёт о создании данного сайта (лабораторная №1).

Настроена современная тема Material for MkDocs, обеспечивающая удобную навигацию, адаптивный дизайн и поддержку подсветки кода. Конфигурационный файл mkdocs.yml позволяет легко расширять структуру и изменять оформление.

Тема Material for MkDocs выбрана благодаря её современному и профессиональному внешнему виду, широким возможностям кастомизации, встроенной поддержке подсветки синтаксиса и удобной навигации, что идеально подходит для представления портфолио с лабораторными работами.

Сайт развёрнут на GitHub Pages из каталога /docs ветки main и доступен по адресу https://username.github.io/portfolio/. Весь процесс разработки отражён в истории коммитов (более пяти осмысленных коммитов), что демонстрирует навыки работы с системой контроля версий Git.
